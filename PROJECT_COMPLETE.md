## ✨ Key Architectural Improvements

### 1. Signal Registry Pattern
```python
_SIGNAL_REGISTRY = []

@register_signal
def signal_1(df): return np.sign(...)
# Auto-discovered, no hardcoding needed
```
**Benefit**: Add/remove signals without changing code

### 2. Parameterized Leverage
```python
run_period(..., long_leverage=1.0)  # or 1.5
# Single engine handles both variants
```
**Benefit**: No code duplication for leverage variants

### 3. Cached Data Loading
```python
_ETF_CACHE = {}  # Session-persistent
# API calls once per session, reuse thereafter
```
**Benefit**: Faster iteration, respects rate limits

### 4. Three Signal Mapping Modes
- **Mode 1**: All signals on all ETFs (ensemble)
- **Mode 2**: Per-ETF signal selection (custom)
- **Mode 3**: Per-signal ETF mapping (original structure)

**Benefit**: Flexible strategy configuration

### 5. Separated Concerns
```
signals.py      → Pure technical analysis
portfolio_engine.py → Backtesting & portfolio construction
Notebooks       → Thin orchestration layer
```
**Benefit**: Testable, reusable, maintainable

## 🚀 How to Use

### Quick Start
```python
from signals import _SIGNAL_REGISTRY
from portfolio_engine import load_price_data, run_period, sharpe

# Load data
price_data = load_price_data(['QQQ', 'SPY', 'TLT'], api_key)

# Run backtest
pnl, pos, ret, ind_pnl = run_period(
    '2020-01-01', '2024-12-31',
    price_data, _SIGNAL_REGISTRY,
    long_leverage=1.5
)

# Calculate metrics
print(f"Sharpe: {sharpe(pnl):.4f}")
```

### Using Refactored Notebook
1. Open `Portfolio_1_0xLong_REFACTORED.ipynb`
2. Set POLYGON_API_KEY in Cell 3
3. Run all cells
4. Check results in Cell 6 (metrics) and Cell 7 (plot)

### Creating New Strategy
1. Copy refactored notebook
2. Modify Cell 3 configuration:
   - Change date ranges
   - Adjust leverage
   - Select specific signals
3. Run notebook
4. Results automatically calculated
